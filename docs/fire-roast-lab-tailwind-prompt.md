# 火候研究所 Fire Roast Lab｜Tailwind CSS 官網開發提示詞

> 目標：依照提供的網站概念圖，使用 **Tailwind CSS** 製作一套靜態官網原型。  
> 開發流程必須採取「分階段完成」：**先完成首頁設計，經確認後，才繼續後續頁面與細節擴充**。  
> 網站圖片素材需優先使用專案 `assets` 資料夾中的既有素材，若同一素材同時有去背與未去背版本，**一律優先使用去背 PNG / WebP**。

---

## 0. 專案角色設定

你是一位資深前端工程師與品牌視覺設計師，請根據我提供的概念圖，建立一個符合「火候研究所 Fire Roast Lab」品牌氣質的靜態網站。

網站風格需要呈現：

- 黑色系高質感餐飲品牌官網
- 烈焰、炭火、研究所、預約制、職人感
- 乾淨、銳利、有層次的視覺排版
- RWD 響應式設計完整，需對應 Desktop / Tablet / Mobile
- 不要做成一般餐廳模板，要保留概念圖中的強烈品牌辨識度

---

## 1. 技術限制與框架要求

### 1.1 技術架構

請使用：

- HTML5
- Tailwind CSS
- 原生 JavaScript
- 不使用 Bootstrap
- 不使用 jQuery
- 不使用 React / Vue / Angular
- 不需要後端功能

### 1.2 Tailwind CSS 使用方式

可使用以下其中一種方式：

1. 若專案已經有 Tailwind 建置流程，請沿用現有設定。
2. 若是純靜態原型，可先使用 Tailwind CDN，但請在註解中標明正式開發時建議改為 CLI / PostCSS 建置。

Tailwind 設定需要包含品牌色：

```js
colors: {
  charcoal: '#0D0D0D',
  flame: '#FF3F00',
  ember: '#FFAA00',
  labWhite: '#F5F5F5',
  techGray: '#8A8A8A'
}
```

## 3. 色彩規範

請完整套用概念圖中的色彩邏輯。

| 顏色名稱 | 色碼 | 使用位置 |
|---|---:|---|
| 深邃炭黑 | `#0D0D0D` | 主要背景色、卡片底色、區塊背景 |
| 烈焰熾橙 | `#FF3F00` | CTA 按鈕、邊框、icon、重點元素 |
| 餘燼金黃 | `#FFAA00` | 標題強調、價格標籤、亮點文字 |
| 實驗室純白 | `#F5F5F5` | 主要文字、Logo 文字、重要資訊 |
| 餘溫科技灰 | `#8A8A8A` | 次要文字、說明文字、輔助線條 |

### 3.1 視覺原則

- 背景以 `#0D0D0D` 為主。
- 橘色 `#FF3F00` 用於行動呼籲與互動狀態。
- 黃色 `#FFAA00` 用於價格、重點標語、強調文字。
- 白色文字不可過亮刺眼，可搭配透明度或灰階降低壓迫感。
- 卡片可使用暗色半透明背景、細橘色邊框與微弱 glow 效果。

## 4. Assets 素材使用規則

### 4.1 資料夾來源

請優先檢查並使用：

```txt
assets/
├── logo/
├── images/
├── icons/
├── backgrounds/
└── qrcode/
```

若現有專案資料夾命名不同，請以實際存在的 `assets` 結構為準，但不要任意刪除素材。

### 4.2 圖片使用優先順序

同一類素材請依照以下優先順序使用：

1. 已去背 PNG / WebP
2. 透明背景 Logo
3. 深色背景版本圖片
4. 原始圖檔

檔名若有以下關鍵字，視為優先素材：

```txt
transparent
alpha
remove-bg
no-bg
去背
透明
logo
hero
chicken
rooster
fire
```

### 4.3 圖片不可任意重製

- 不要重新繪製 Logo。
- 不要改變品牌名稱圖片的文字。
- 不要改變雞 Logo 的比例與造型。
- 不要改色既有 Logo，除非素材本身已有對應版本。
- 不要使用外部圖片取代 assets 中已有素材。

### 4.4 缺少素材時的處理方式

若找不到合適素材：

- 先使用合理的 placeholder 區塊。
- 在 HTML 註解中標記需要補上的素材名稱。
- 不要自行生成全新品牌 Logo。

範例：

```html
<!-- TODO: assets/images/hero-chicken-transparent.png 尚未提供，請補上去背主視覺雞圖 -->
```

## 7. Header / Navbar

### 7.1 Desktop / Tablet

Header 需符合概念圖：

- 左側放 Logo 與品牌名稱圖片或文字
- 右側放導覽選單
- 導覽選單靠右排列
- 整體固定在頁面上方或維持 sticky 效果
- 背景為黑色或半透明黑色
- 使用細橘色下緣或邊框點綴

導覽項目：

```txt
首頁
品牌介紹
爐烤介紹
火候建議
預約方式
聯絡我們
```

### 7.2 Active 狀態

首頁 active 狀態：

- 文字使用 `#FFAA00` 或 `#FF3F00`
- 底部加短橘色底線
- 不要太粗，需精緻

### 7.3 Mobile

Mobile 寬度約 390px 以下時：

- 左側顯示 Logo
- 右側顯示 hamburger icon
- 點擊後展開手機選單
- 選單背景為深黑色
- 選單文字需大且易點擊
- 展開動畫簡潔即可

## 8. Hero 主視覺區

### 8.1 Desktop 版

Hero 需採左右雙欄：

左側：

- 大標題：火候研究所
- 副標題：預約制 楓烤全雞
- 介紹文字：

```txt
我們專注於多元化方式烤烤。
透過科學與人工控火候烘出濃郁的完美！
```

> 若覺得文字略不自然，可保留概念圖精神，微調為更通順版本，但不要改變品牌語氣。

右側：

- 使用 assets 中的雞 Logo / hero 主視覺圖
- 若有火焰背景圖，可放在雞圖後方
- 主圖需有烈焰、橘紅、發光感
- 不能讓圖片壓過文字可讀性

### 8.2 Hero 按鈕

Hero 下方需有兩個 CTA：

1. 立即預約
2. 加入我們

按鈕設計：

- 「立即預約」使用烈焰熾橙背景 `#FF3F00`
- 「加入我們」使用透明底 + 白 / 灰框
- hover 時加亮、微微上移或 glow

### 8.3 社群 icon

Hero CTA 旁需放社群 icon：

- Facebook
- Instagram
- LINE

若 assets 中有 icon，請優先使用。  
若沒有，可用文字或簡單圓形 icon placeholder，但需標記 TODO。

### 8.4 Tablet 版

Tablet 約 768px ~ 1024px：

- 仍可使用雙欄
- Hero 圖稍微縮小
- 文字大小比 Desktop 略小
- 卡片間距縮小

### 8.5 Mobile 版

Mobile 約 390px 以下：

- Hero 改為單欄
- Logo / 標題 / CTA 置中或偏左，但需清楚
- 主圖可以移到標題下方或保留背景感
- CTA 按鈕滿版或接近滿版
- 社群 icon 置中排列

## 13. RWD 響應式要求

請依照概念圖規劃三種主要斷點：

| 裝置 | 寬度參考 | 設計重點 |
|---|---:|---|
| Desktop | 1440px 以上 | 雙欄 Hero、多欄卡片、資訊完整呈現 |
| Tablet | 768px ~ 1024px | 壓縮間距，保留雙欄或較緊湊排列 |
| Mobile | 390px 以下 | 單欄排列，CTA 易點擊，導覽收合 |

### 13.1 RWD 原則

- 優先確保 Mobile 可讀性。
- 不要讓圖片擠壓文字。
- 不要讓卡片在小螢幕過窄。
- CTA 按鈕高度至少 44px。
- 文字行高需舒適。
- 保留概念圖中的黑橘品牌感。

## 14. 視覺細節要求

### 14.1 背景

- 整站背景為 `#0D0D0D`
- 可加入非常淡的火焰粒子、暗紅光暈、radial-gradient
- 不可讓背景過花，影響閱讀

### 14.2 邊框與 Glow

可使用：

- `border: 1px solid rgba(255, 63, 0, 0.45)`
- `box-shadow: 0 0 20px rgba(255, 63, 0, 0.12)`
- hover 時略微加亮

### 14.3 字體

若無指定字體，請使用系統字體：

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans TC", sans-serif;
```

品牌標題可使用較有筆刷感或襯線感的字體，但不要引入不穩定外部字體。若使用 Google Fonts，需註明可改成本機字體。
